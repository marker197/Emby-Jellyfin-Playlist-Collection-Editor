Connecting to Emby

03
1
Enter your Server URL

The full address of your Emby server including port — e.g. http://192.168.1.10:8096. No trailing slash. Use https:// if your server has SSL enabled.

2
Enter your Emby username and password

Your standard Emby account credentials. Use the eye icon to toggle password visibility.

3
Click Connect

The app authenticates, loads your movie libraries and playlists, and opens the main panel. A green dot confirms you are connected. Settings are saved to browser storage so they persist between sessions.

To disconnect, click the Disconnect button in the top-right corner of the main panel.

Smart First-Run Detection

04 New

On the very first login from any device, the app automatically scans your library and works out which franchises you have substantial coverage of. A welcome overlay then suggests playlists you could create in one click — no manual searching needed.

How it works

After login, the app calls Emby once to pull every movie in your selected libraries, then runs every built-in franchise against the title-matching engine. Any franchise where you have at least 3 movies present AND at least 50% of the franchise list in your library is shown as a recommendation.

1
Welcome overlay appears

On your first successful login, a dark overlay shows a list of detected franchises with how many films of each you already own (e.g. "Marvel Cinematic Universe — 28 of 33 films in your library").

2
Tick the franchises you want

Click any row to toggle it on. Pick as many or as few as you like. Up to 12 best matches are shown.

3
Click Create Selected

The app creates each playlist sequentially in chronological order using the franchise watch order. Progress is shown per franchise.

✅
Only shows once The welcome overlay only appears on the very first successful login. Once dismissed (via Skip or Create Selected) it never reappears, even on subsequent logins. You can always create the same playlists later from the normal search panel.
💡
Quick start tip If you're new and have a large library, just let the app pick everything for you — tick all suggestions, hit Create Selected, and you'll have a dozen perfectly ordered franchise playlists in under a minute.

Franchise Source Toggle — Built-in vs TMDB Order

07 New

Both the Create Playlist and Refresh Playlist sections have a Movie order from toggle that controls which franchise list is used for ordering the movies in your playlist.

The two modes

ModeWhat it usesWhen to use it
Built-in / Uploaded The built-in franchise list, or your custom override / imported CSV version if one exists Default — best for most situations, uses carefully curated watch orders
TMDB Order The list fetched from TMDB the last time you ran a TMDB refresh When you want the very latest TMDB-sourced film order, e.g. to pick up a newly released sequel automatically

How it works

When you select TMDB Order and a franchise is chosen, the app looks up the TMDB-sourced list for that franchise. If no TMDB data has been fetched yet for it, the app falls back to the built-in list and shows a warning prompting you to run a TMDB refresh first.

💡
TMDB Order and TMDB refresh modes work together If you ran a TMDB refresh in Override mode, the built-in list is already updated with TMDB data — both toggle options give the same result. The TMDB Order toggle is most useful when you ran TMDB refresh in Create separate TMDB copy mode, so the two versions (built-in and TMDB) coexist and you can switch between them.
✅
The toggle is independent per section The Create section and Refresh section each have their own toggle. Changing one doesn't affect the other.

Preview Before Creating

The Preview button lets you dry-run any playlist creation without writing anything to Emby. It shows exactly which library movie will be picked for each franchise title, so you can spot mismatches and fix them before creating the playlist.

How to use it

2

3

4

Creating a Playlist from a Collection

10 New

If you already have an Emby collection containing the movies you want, you can use it as the source instead of scanning your whole library. This is faster, more targeted, and useful when you have manually curated a collection in Emby.

What is an Emby collection?

An Emby collection (called a BoxSet internally) is a named group of movies you create in Emby by right-clicking films and choosing Add to Collection. Collections appear in your Emby dashboard under the Collections view. The app reads these directly from your server.

1
Select a franchise and open the Create panel

Search for a franchise and click it to open the Create Emby Playlist panel below the search results.

2
Click Use Collection

The source toggle at the top of the Create panel has two options. Click Use Collection — it highlights blue and a collection dropdown appears below the toggle.

3
Select your collection

The dropdown lists all Emby collections with their item counts. Choose the one that contains the franchise movies. Click the circular reload button beside the dropdown to refresh the list from Emby if needed.

4
Set remaining options and click Create Playlist

Choose user, sort order and playlist name as normal, then click Create. The app fetches only the movies inside the selected collection and matches them against the franchise watch order. All the same features apply — green pills for matched films, red clickable pills for anything missing, and the Send All Radarr bar if needed.

Library scan vs collection — when to use each

Use caseBest source mode
First time creating a franchise playlistScan Library — finds everything you have across all libraries
You have a hand-picked Emby collectionUse Collection — faster, only uses films you've already curated
Multiple libraries (4K + HD)Scan Library with both libraries checked
You want a playlist from a specific setUse Collection — gives you precise control over source films
No collections created yet in EmbyScan Library — collections must be created in Emby first
✅
Collections are pre-loaded on login The collection list is fetched from Emby in the background as soon as you connect, so the dropdown is ready the moment you switch to collection mode. Use the reload button to pick up any collections created in Emby after you logged in.
💡
The source mode persists between franchise selections If you switch to collection mode to create one playlist, it stays in collection mode when you select the next franchise — you don't need to toggle it again. Switch back to Scan Library at any time by clicking the Library button.

Refreshing an Existing Playlist

11 Updated

Refresh does two things in a single operation: it adds any newly downloaded franchise movies that have appeared in your library since the playlist was created, then re-sorts the entire playlist into the correct chronological order.

1
Select a playlist from the dropdown

All your Emby playlists are listed. Click the circular arrow button beside the dropdown to reload the list from Emby at any time.

2
Choose a franchise for ordering

Select the franchise whose watch order you want to apply. All built-in franchises plus any TMDB-sourced or custom/imported ones are available. Built-in entries have no badge, TMDB entries are labelled [tmdb], and custom ones show a ★ marker.

3
Choose your movie order source

A Movie order from toggle below the franchise dropdown lets you pick Built-in / Uploaded (default) or TMDB Order. This controls which version of the franchise list drives the ordering — see Franchise Source Toggle for full details.

4
Click Refresh Playlist

The app scans your checked libraries for new franchise movies not yet in the playlist, adds them, then moves every item to its correct position. A progress bar shows the current movie being moved.

✅
Use Refresh after a Radarr download completes When Radarr finishes downloading a requested movie and Emby scans it in, click Refresh to slot it automatically into the right position without recreating the playlist.

Any items in the playlist that don't match the franchise list are kept and sorted by production year, placed at the end after all matched titles.

Franchise Source Labels

13 New

Every franchise in the app is now labelled with its source so you always know where the movie list came from. Labels appear in the franchise manager list, on franchise cards in search results, and in the refresh dropdown.

LabelColourMeaning
built-in Purple Part of the app's original built-in franchise database — pre-loaded, no action needed
tmdb tmdb Fetched from TMDB via the TMDB refresh — contains the latest film list from The Movie Database. These are stored separately from both built-in and custom lists
custom Green Created by you manually, imported from CSV, or saved as an override of a built-in list — stored in your browser and persists between sessions

Priority order

When the app looks up a franchise's movie list for playlist creation or refresh, it uses this priority: TMDB data first (if in TMDB mode) → custom override → built-in. In built-in mode, custom overrides always take precedence over the original built-in data.

✅
TMDB and custom lists are kept separate Running a TMDB refresh never touches your custom franchise edits, and saving a custom franchise never touches your TMDB data. They live in different storage keys and the label always tells you which one you're looking at.

Conflict Resolution — Naming Clashes with Built-in Franchises

16 New

When you save a franchise — either manually via the New form or by importing a CSV — and the name matches an existing built-in or tmdb franchise, the app pauses and shows a conflict resolution modal so you can decide what happens instead of silently overwriting data.

The two options

OptionWhat it doesWhen to choose it
Override built-in list Your new version replaces the existing franchise for all future playlist operations. The original built-in data is no longer used. The franchise is labelled custom. Your version is more accurate or complete than the built-in one, and you want everything to use it from now on
Save as custom version Your version is saved under "Franchise Name (Custom)" alongside the original. Both coexist — the built-in keeps its name and your version has the (Custom) suffix. Labelled custom. You want to keep the original intact and have a separate variant — e.g. a different watch order preference or regional title set

Cancelling

If you click Cancel, nothing is saved and no data is changed.

CSV import with multiple conflicts

When importing a CSV that contains several franchises matching built-in names, a conflict modal is shown for each one in sequence. Franchises with unique names (no conflict) are saved immediately without interruption.

💡
Restore a built-in list you've overridden Open the Manage Franchises panel, find the franchise, click it to open the editor, and use the Delete Franchise button. This removes your custom override and restores the original built-in data.

CSV format

The file must have three columns per row: Franchise Name, Movie Title, and Release Year. One movie per row. A header row is automatically detected and skipped.

TMDB Metadata Enrichment

18 New

Optional TMDB (The Movie Database) integration keeps your franchise lists current. With an API key configured, the app can fetch up-to-date franchise data so newly released films appear in your watch orders without manual editing.

Get a free TMDB API key

1
Create a TMDB account

Sign up free at themoviedb.org.

2
Request an API key

Go to Settings → API in your TMDB account. Request a v3 API key (the standard one). Approval is usually instant for personal use.

3
Paste the key into the app

Scroll to the TMDB section below the Radarr servers, paste your key into the field, and click Test connection. A green confirmation appears if the key is valid.

4
Choose a refresh mode and click Refresh franchises from TMDB

Select Override built-in list or Create separate TMDB copy from the toggle above the refresh button — see TMDB Refresh Modes below. Then click the button. The app walks through every built-in franchise with a known TMDB Collection mapping, fetches the latest film list, and saves it. Progress is shown per franchise.

What gets fetched

The app contains a built-in mapping of 70+ franchise names to their TMDB Collection IDs. When you click Refresh, each one is fetched and the movie list is replaced with current TMDB data, sorted by release date. Newly released films are automatically included (e.g. the next MCU entry when it joins the collection).

✅
Your API key never leaves your network The key is stored in browser local storage and only sent over HTTPS directly to TMDB's API. The app itself does not transmit it anywhere else.

TMDB Refresh Modes

19 New

Before running a TMDB refresh, a When refreshing from TMDB toggle in the TMDB panel lets you choose what happens to your existing franchise data.

Mode 1 — Override built-in list

TMDB data replaces the matching built-in franchise. The result is labelled tmdb and takes priority over the built-in list. Both the Create and Refresh playlist source toggles in Built-in / Uploaded mode will then automatically use the TMDB data because it has replaced the original.

Mode 2 — Create separate TMDB copy

Built-in lists are left completely untouched. TMDB data is saved as a parallel entry named "Franchise Name (TMDB)". Both versions coexist in the franchise list — the original built-in and the new TMDB copy. To use the TMDB version when creating or refreshing a playlist, select TMDB Order in the source toggle, or search for and pick the (TMDB) variant directly.

Comparison

Override built-inCreate separate copy
Built-in list changed?Yes — replaced by TMDB dataNo — untouched
New entry created?No — same name, new contentYes — "Franchise (TMDB)" added
Label showntmdbtmdb
Best forYou always want the latest TMDB data and don't need the originalYou want to compare both versions or switch between them per playlist
Custom edits preserved?TMDB replaces any existing custom override tooCustom overrides are unaffected
⚠️
Override mode replaces custom edits too If you've manually edited a built-in franchise and then run a TMDB refresh in Override mode, the TMDB data replaces your custom edits for that franchise. Export your custom franchises to CSV first if you want a backup, or use Create separate copy mode to keep your edits safe.
💡
Re-running a TMDB refresh You can run the TMDB refresh as often as you like. In Override mode it simply updates the tmdb-labelled entries with the latest data. In Create copy mode it refreshes the existing (TMDB) entries, or creates them if they don't exist yet.

Radarr Auto-Reconnect

21 New

Once you connect to Emby, the app automatically monitors all configured Radarr servers in the background. No manual action is required.

How it works

Every 60 seconds the app silently pings each Radarr server's status endpoint. The result is reflected immediately in two places:

WhereWhat you see
Radarr section title● live in green when all servers are reachable; ● reconnecting… in grey when any server is offline
Individual server status lineGreen "Connected — Radarr v…" when online; red "Offline — retrying…" when not reachable

When a server that was offline comes back up, the status updates to green on the next 60-second ping automatically — no page reload or manual test needed.

💡
Each ping has an 8-second timeout A slow or unresponsive Radarr server will time out after 8 seconds and be marked offline, without blocking the rest of the UI or the other server pings.

  • A Send All Missing to Radarr button
1
Choose a Radarr server from the dropdown

The dropdown lists your configured servers by their label names. Select the one you want to send all requests to.

2
Click Send All Missing to Radarr

How Title Matching Works

29 New

The matching engine compares franchise title list entries against your Emby library. It was completely rewritten to eliminate false positives while still handling common title variations.

Matching rules (in order)

1
Exact normalised match

Punctuation, case and extra spaces are stripped from both titles and compared character-for-character. This handles the vast majority of matches cleanly — e.g. "Alien: Covenant" matches "Alien Covenant".

2
Significant word matching with sequel guard

Stop words (the, a, an, of, and…) are removed. Numbers are kept — "Scream" and "Scream 2" are treated as different titles because the number is a meaningful distinguishing word.

3
The library entry must not have more words than the franchise title

This is the key rule that prevents false positives. "Star Trek" (2 words) will never match "Star Trek Into Darkness" (4 words). "John Wick" (2 words) will never match "John Wick: Chapter 2" (3 words). A short title cannot match a longer one.

Examples

Franchise titleLibrary titleResult
AliensAliens✓ Match — exact
Alien: CovenantAlien Covenant✓ Match — punctuation stripped
AlienAliens✗ No match — "aliens" has more words
Scream 2Scream 2✓ Match — exact with number
ScreamScream 2✗ No match — sequel guard
Star TrekStar Trek Into Darkness✗ No match — library title has more words
RVMarvel✗ No match — "rv" is one word under 5 chars, exact only
Doctor StrangeDoctor Strange in the Multiverse of Madness✗ No match — library title has more sig. words

What to do if a film still doesn't match

IssueFix
Regional title variantEdit the franchise list entry to use the title Emby shows — e.g. Philosopher's Stone vs Sorcerer's Stone

The playlist isn't reordering correctly

Each move is sent to Emby one at a time. For large playlists (50+ items) this takes a moment. Wait for the progress bar to reach 100% before checking in Emby. You may also need to refresh the Emby client to see the updated order.

Data lost after closing the browser

All settings, Radarr servers and custom/imported franchises are stored in browser local storage tied to the URL you use to open the file. Use the file from a consistent location each time.

💡
Bookmark the file for reliable persistence Serve it with python3 -m http.server 8080 and bookmark http://localhost:8080/emby-chrono-playlist.html. The URL stays constant so all local storage data persists reliably.

CSV import shows no franchises found

Check that the file uses commas as delimiters and has at least a Franchise column and a Movie Title column. If your CSV editor uses semicolons, resave with comma delimiters. The app handles RTF files automatically — if you see garbage characters, try saving as plain .csv from your spreadsheet app instead.

TMDB refresh updates nothing

Make sure your TMDB API key is entered and the Test connection button shows a green success message first. If a franchise shows as "failed" in the status, it usually means the TMDB collection ID has changed — this is rare but can happen when TMDB reorganises its data.

I chose the wrong option in the conflict modal

If you overrode a built-in list by mistake, open the Manage Franchises panel, click the franchise, and use the Delete Franchise button to remove your custom override. The original built-in data will reappear. If you saved a custom copy under the wrong name, delete it and re-import or re-add it.

TMDB Order toggle shows a warning

The warning "No TMDB data found" means you haven't run a TMDB refresh yet, or the selected franchise doesn't have a TMDB Collection mapping in the app. Run a TMDB refresh first, or switch back to Built-in / Uploaded mode.

Jellyfin & Plex Compatibility

32 New

This app is purpose-built for Emby and uses Emby-specific APIs. Here's what to know if you're interested in using it with other media servers:

Jellyfin

Jellyfin is Emby's open-source fork. Its API is nearly identical to Emby's, so compatibility is very high. Many Jellyfin users have successfully adapted this app:

  • Server URL: Works the same as Emby (default port 8096)
  • Playlists: Full compatibility — Jellyfin playlists use the same endpoint
  • Collections: Full compatibility — BoxSets API is identical
  • Libraries: Compatible, though library structure may differ slightly
  • Known differences: Some metadata fields may not exist or have different values; Jellyfin's user permissions system works slightly differently
ℹ️
Jellyfin is likely to work with minimal or no changes. Try it — most features should work out of the box. If you encounter errors, check that your Jellyfin API endpoint matches the app's expectations.

Plex

Plex is fundamentally different from Emby and Jellyfin. Its API uses a completely different architecture, authentication method, and data structures. This app is not compatible with Plex without significant rewrites:

  • Authentication: Plex uses token-based OAuth, Emby uses username/password
  • Endpoints: Plex uses /library/sections, /playlists, etc. — completely different from Emby's /Items and /Playlists
  • Playlists: Plex playlists have a different structure and creation workflow
  • Collections: Plex doesn't have a direct equivalent to Emby's BoxSets
  • Library scanning: Plex doesn't expose real-time library access the same way Emby does
❌
Plex would require a complete rewrite. The app's entire architecture is built around Emby's API patterns. While technically possible, adapting it to Plex would essentially mean building a new app from scratch.

Other Media Servers (Kaleidescape, MediaServer, etc.)

Similar to Plex — each media server has its own proprietary API and architecture. This app cannot be adapted to work with them without substantial changes to the codebase.

How to Try on Jellyfin

If you have a Jellyfin server, try the app as-is:

  1. Open the app and enter your Jellyfin server URL (usually http://192.168.1.x:8096)
  2. Use your Jellyfin username and password
  3. If login works, the core features should work. Radarr integration will work if configured the same way.
  4. If you encounter errors, the Network Activity panel (bottom of the app) will show failed requests — note the specific API endpoint that fails.
💡
Report Jellyfin issues on GitHub If you're a Jellyfin user and discover bugs or incompatibilities, open an issue with the specific API error messages. Fixes for Jellyfin compatibility may be possible if the differences are minor.
